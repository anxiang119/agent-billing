## 1. 系统概述

智能体计费系统是一个面向智能体服务的资源计费平台，支持 Token、OSS 存储、外部 API 调用等多种资源类型的计费。系统提供包月计费和按量计费两种模式，支持阶梯计费、优惠活动、预算预警、账单生成、发票管理等功能。

### 1.1 核心特性

- **多资源类型计费**：支持 Token、OSS、外部 API 等多种资源
- **灵活计费模式**：支持包月计费和按量计费
- **阶梯计费**：支持多阶段阶梯定价
- **优惠管理**：支持满减优惠，可按资源类型和租户配置
- **预算预警**：支持用户和租户级别的预算设置和邮件预警
- **自动账单**：每月自动生成账单，支持用户维度和租户维度汇总
- **发票管理**：支持电子发票开具和下载
- **预付费模式**：支持充值、预扣、余额管理

### 1.2 系统角色

| 角色 | 描述 |
|------|------|
| 运营人员 | 配置定价规则、创建优惠活动、代开发票 |
| 租户管理员 | 管理租户用户、设置预算、查看账单、申请发票 |
| 用户 | 使用智能体服务、充值、查看消费 |

| status | TINYINT | 状态(0:禁用,1:启用) | DEFAULT 1 |
| credit_limit | DECIMAL(12,2) | 信用额度 | DEFAULT 0 |
| created_at | DATETIME | 创建时间 | NOT NULL |
| updated_at | DATETIME | 更新时间 | NOT NULL |

| 字段名 | 类型 | 说明 | 约束 |

#### 账户表 (account)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|

### 6.1 定价配置管理模块

#### 6.1.1 模块功能描述

定价配置管理模块负责管理各种资源类型的定价规则，包括 Token、OSS、外部 API 的计费策略。运营人员可以创建、修改、删除定价配置，支持包月计费和按量计费两种模式，支持阶梯计费规则。

#### 6.1.2 核心业务处理流程

**流程1：创建 Token 定价配置**

```
1. 接收运营人员提交的定价配置请求
2. 验证请求参数
   - 资源类型必须为 TOKEN
   - 计费模式必须为 MONTHLY 或 USAGE
   - 若为 USAGE，必须配置单价
   - 若配置阶梯，验证阶梯范围连续性
3. 检查是否与现有配置冲突
4. 生成定价配置记录
5. 保存定价配置到数据库
6. 保存定价配置历史记录
7. 清除相关缓存
8. 返回配置ID
```

**流程2：获取生效定价配置**

```
1. 接收查询请求(用户ID、资源类型、模型)
2. 查询系统级默认配置
3. 查询租户级配置(如果有)
4. 按优先级排序(租户 > 系统)
5. 检查配置有效性
   - 当前时间在有效期内
   - 配置状态为启用
6. 返回最高优先级的配置
```

**流程3：修改定价配置**

```
1. 接收修改请求
2. 验证权限(运营人员)
3. 查询原配置
4. 验证新配置参数
5. 保存修改前的配置到历史表
6. 更新定价配置
7. 清除缓存
8. 记录审计日志
```

### 6.2 计费引擎模块

#### 6.2.1 模块功能描述

计费引擎模块是系统的核心模块，负责智能体调用的费用计算、预扣、结算。模块支持多种资源类型的计费，支持阶梯计费和优惠应用，确保计费的准确性和一致性。

#### 6.2.2 核心业务处理流程

**流程1：智能体调用预估与预扣**

```
1. 接收智能体调用请求
2. 验证用户状态和余额
3. 预估本次调用的资源消耗
   - Token：根据模型预估输入/输出量
   - OSS：预估流量和请求次数
   - 外部API：预估调用次数
4. 获取生效的定价配置
5. 计算预估费用
   - 应用阶梯计费规则(如适用)
   - 按资源类型分别计算
6. 检查余额是否充足
   - 若不足，返回 INSUFFICIENT_BALANCE 错误
7. 生成预扣记录
8. 执行预扣操作(分布式锁保证原子性)
   - 检查账户版本号
   - 扣减余额
   - 增加冻结金额
   - 更新版本号
9. 返回预扣ID和预估费用
```

**流程2：智能体调用完成结算**

```
1. 接收智能体调用完成通知
2. 获取实际资源消耗数据
3. 获取预扣记录
4. 获取生效定价配置
5. 计算实际费用
   - Token费用 = 输入token单价×输入量 + 输出token单价×输出量
   - 应用阶梯计费规则
   - OSS费用 = 流量费用 + 请求费用
   - 外部API费用 = 调用次数×单价
   - 总费用 = 各资源费用之和
6. 查询并应用优惠规则
7. 计算最终费用 = 实际费用 - 优惠金额
8. 对比预扣金额与最终金额
9. 执行结算操作(分布式锁)
   - 若预扣 > 实际：退还差额
   - 若预扣 < 实际：补扣差额(余额不足时记录差额不扣)
   - 减少冻结金额
10. 创建消费记录
11. 更新用户和租户累计消费
12. 更新预扣记录状态为已结算
13. 触发预算预警检查
14. 返回结算结果
```

**流程3：阶梯计费计算**

```
输入：使用量、阶梯规则
输出：费用

算法：
1. 按阶梯下限升序排序
2. 剩余使用量 = 总使用量
3. 总费用 = 0
4. 遍历每个阶梯：
   a. 当前阶梯可用量 = min(阶梯上限 - 阶梯下限, 剩余使用量)
   b. 当前阶梯费用 = 当前阶梯可用量 × 阶梯单价
   c. 总费用 += 当前阶梯费用
   d. 剩余使用量 -= 当前阶梯可用量
   e. 若 剩余使用量 == 0，跳出循环
5. 返回 总费用
```

### 6.3 账户与余额管理模块

#### 6.3.1 模块功能描述

账户与余额管理模块负责用户账户的创建、余额管理、充值、退款等操作。模块使用分布式锁和乐观锁保证余额操作的一致性，支持余额不足时的处理机制。

#### 6.3.2 核心业务处理流程

**流程1：充值操作**

```
1. 用户发起充值请求
2. 验证用户状态
3. 生成充值记录(状态为待支付)
4. 调用支付网关创建支付订单
5. 返回支付信息(如支付二维码、跳转URL)
6. 用户完成支付
7. 支付网关异步通知支付结果
8. 验证支付通知签名
9. 更新充值记录状态为已支付
10. 增加用户余额(分布式锁)
11. 记录充值流水
12. 返回充值成功
```

**流程2：余额扣减**

```
输入：用户ID、扣减金额

1. 获取用户账户信息
2. 检查余额是否充足
   - 若不足，返回错误
3. 使用Redis分布式锁(锁键：account:user:{user_id})
4. 获取锁后重新查询账户(乐观锁)
5. 验证余额和版本号
6. 更新账户
   - balance -= 扣减金额
   - frozen_amount += 扣减金额
   - version += 1
7. 释放锁
8. 返回成功
```

**流程3：余额充值**

```
输入：用户ID、充值金额

1. 使用Redis分布式锁
2. 获取锁后查询账户
3. 更新账户
   - balance += 充值金额
   - total_recharge += 充值金额
   - version += 1
4. 释放锁
5. 返回成功
```

#### 6.4.2 核心业务处理流程

**流程1：创建消费记录**

   - 预扣记录ID

3. 写入MySQL消费记录表
4. 异步写入Elasticsearch(用于快速查询)

6. 返回记录ID
```

   - 缓存键：monthly:consume:{tenant_id}:{user_id}:{yyyy-MM}

4.聚合计算当月消费

**流程3：查询历史消费**

```

   - 用户ID、时间范围

### 6.5 优惠活动管理模块

#### 6.5.1 模块功能描述

优惠活动管理模块负责创建和管理满减优惠活动，支持按资源类型和租户范围配置优惠。模块自动检测用户消费是否满足优惠条件，计算优惠减免金额。

#### 6.5.2 核心业务处理流程

**流程1：创建优惠活动**

```
1. 接收优惠配置
2. 验证参数
   - 优惠名称必填
   - 优惠类型必填
   - 至少选择一个资源类型
   - 满减规则格式正确
3. 生成优惠编码
4. 保存优惠配置到数据库
5. 清除优惠缓存
6. 返回优惠ID
```

**流程2：计算优惠减免**

```
输入：用户ID、租户ID、消费金额、资源类型、计费月份

1. 查询所有生效的优惠活动
2. 筛选适用的优惠
   - 当前时间在有效期内
   - 资源类型匹配
   - 租户在适用范围内(或全部租户)
3. 计算各优惠的减免金额
   - 遍历优惠规则的满减阶梯
   - 找到满足条件的最高阶梯
   - 获取对应减免金额
4. 应用优惠策略
   - 若多个优惠同时满足，应用最优优惠
   - 检查用户月度优惠使用次数限制
5. 返回减免金额和优惠ID
```

**流程3：应用优惠到账单**

```
1. 获取账单的消费明细
2. 按资源类型汇总消费金额
3. 遍历每种资源类型
4. 计算该资源类型的优惠减免
5. 累加总减免金额
6. 更新账单的减免金额
7. 创建用户优惠记录
8. 返回优惠应用结果
```

### 6.6 预算与预警模块

#### 6.6.1 模块功能描述

预算与预警模块负责管理用户和租户的消费预算，在消费达到预警阈值或预算上限时发送预警通知或停止服务。

#### 6.6.2 核心业务处理流程

**流程1：设置用户预算**

```
1. 接收预算配置
2. 验证参数
   - 预算上限为正数
   - 预警阈值为0-100
3. 查询用户现有预算
4. 更新或创建预算配置
5. 返回成功
```

**流程2：检查预算预警**

```
1. 获取用户预算配置
2. 若无预算配置，跳过
3. 获取当月累计消费
4. 计算消费占比 = 累计消费 / 预算上限
5. 遍历预警阈值(80%, 90%)
6. 检查是否已发送过该等级预警
7. 若消费占比 >= 阈值且未发送预警：
   a. 创建预警记录
   b. 发送邮件通知给用户和管理员
   c. 标记已发送
8. 检查是否超出预算上限
9. 若超出：
   a. 停止用户服务
   b. 更新服务状态为已停止
   c. 发送停止通知
```

**流程3：恢复用户服务**

```
1. 管理员手动恢复或预算调整后自动恢复
2. 更新用户服务状态为正常
3. 清除预警标记
4. 发送恢复通知
```
